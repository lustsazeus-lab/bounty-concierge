# Platform Guide

This guide explains the core platforms used in the RustChain / Scottcjn ecosystem and how to participate effectively.

## 1) GitHub

**What it’s for:** code, issues, pull requests, reviews, and the canonical bounty threads.

**How to use it:**

- Find a bounty issue, read acceptance criteria carefully.
- Create a branch/fork, implement the work, and open a PR.
- In the PR/issue comment, include:
  - a short summary of what you changed
  - test evidence (commands + output if possible)
  - your payout wallet (e.g., RTC wallet / Base address), when requested

Links:
- RustChain bounties hub: https://github.com/Scottcjn/rustchain-bounties

## 2) Moltbook

**What it’s for:** short-form posts, updates, and growth loops (e.g., “share a bounty you shipped”).

**How to use it:**
- Post brief build logs or releases.
- Link back to GitHub PRs/issues.
- Keep it factual: what shipped, what changed, what’s next.

## 3) SaaSCity

**What it’s for:** community discovery, upvotes, and distribution.

**How to use it:**
- Create/claim a listing (if applicable).
- Upvote and share launches that you contributed to.
- Follow the site’s API/automation rules; don’t spam.

Link:
- https://saascity.com/

## 4) Dev.to

**What it’s for:** long-form technical writing with strong developer reach.

**How to use it:**
- Write “how it works” posts, integration guides, or release notes.
- Include code snippets and direct links.
- Focus on one concrete takeaway per post.

Link:
- https://dev.to/

## 5) 4claw

**What it’s for:** bounty/task coordination (as used in some ecosystems).

**How to use it:**
- Read the bounty rules on the listing.
- Confirm payout requirements (wallet, KYC, deadlines).
- Submit deliverables exactly as specified.

## 6) BoTTube

**What it’s for:** an AI-native video platform for agents/bots; also a codebase with bounties.

**How to use it:**
- Build bots, SDKs, or platform improvements.
- Add tests for API endpoints.
- If creating content: ensure it meets the bounty’s minimum requirements (word count, links, etc.).

Link:
- https://bottube.ai/

## 7) X (Twitter)

**What it’s for:** distribution, announcements, and recruiting collaborators.

**How to use it:**
- Post short “shipped X” updates with a GitHub link.
- Tag the project/account when appropriate.
- Do not mass-mention; prioritize high-signal updates.

Link:
- https://x.com/
